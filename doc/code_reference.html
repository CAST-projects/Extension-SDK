get_annotations()¶

Returns the annotations associated with the object.

A list of annotations is returned.

Return type:list of tuple (cast.analysers.Type,dict)

An annotation is a tuple (type,parameters) where:

  • type is a Type object
  • parameters is a dictionary with the actual parameters

Add a violation for the given rule.

rule is the name of a Metamodel property

Parameters:

  • bookmark – a bookmark or a list of bookmarks to indicate position of the violation
  • additional_bookmarks – additional bookmarks that help understanding the violation

External link API¶

API for External Links.

External Links allow to retreive objects from other analysis marked as dependency.

For example one can retreive database Tables, View etc...

cast.analysers.external_link.find_objects(name, category_name)¶

Find all the external objects of a given name and category.

Parameters:
  • name – name of the object
  • category_name – name of a metamodel category
Return type:

list of cast.analysers.ExternalObject

Example :

# find all tables named ‘Author’:

tables = find_objects(‘Author’,’Database Table’)

@rtype: [cast.analysers.ExternalObject]

Tests API Reference¶

This module provide testing help for developping plugins.

Simple usage:

import cast.analysers.test

analysis = cast.analysers.test.DotNetTestAnalysis()

analysis.add_selection(‘path to .csproj’)

analysis.run()

class cast.analysers.test.TestAnalysis(technology)¶

Base class for all tests analysis.

add_selection(path)¶

Select some path to analyze.

Parameters:path – selection path to be added. String.
add_database_table(table_name, technology)¶

Register a database table as an dependency.

Parameters:
  • table_name – table name to be added. String.
  • technology – technology name of the table. Possible values are : ‘Oracle’, ‘TSQL’. String.
set_verbose()¶

Will print log to console

run()¶
